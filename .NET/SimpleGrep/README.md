# SimpleGrep 取扱説明書

SimpleGrep は、指定したフォルダ配下のテキストファイルを grep 検索し、検索結果を一覧表示する Windows Forms ツールです。

## 起動

`SimpleGrep.exe` を起動します。

検索履歴やチェック状態などの設定は、実行ファイルと同じフォルダの `SimpleGrep.settings.json` に保存されます。

## 基本的な検索手順

1. `検索フォルダパス` に検索対象フォルダを入力します。
   `参照` ボタン、またはフォルダ/ファイルのドラッグ＆ドロップでも指定できます。
2. `対象ファイル` に検索対象のファイルパターンを入力します。
   例: `*.cs`, `*.txt`, `*.cs/*.java`
3. `検索キーワード` に検索したい文字列を入力します。
4. 必要に応じて検索オプションを設定します。
5. `検索` ボタンを押します。

検索結果は下部の一覧に表示されます。検索中は進捗率と経過時間が表示され、`中止` ボタンで検索をキャンセルできます。

## 検索条件

### 検索フォルダパス

検索を開始するフォルダです。

`サブフォルダも対象` をオンにすると、配下のサブフォルダも検索します。

### 対象ファイル

検索するファイル名パターンです。
複数指定する場合は `/` 区切りで入力します。

例:

```text
*.cs
*.java
*.txt
*.cs/*.java
```

### 検索キーワード

検索する文字列です。

`正規表現` をオンにすると、検索キーワードを正規表現として扱います。

### 複数キーワード検索

`複数` ボタンを押すと、複数キーワード入力画面を開きます。

1行に1キーワードずつ入力し、OKすると複数キーワードで検索します。

### 除外フォルダ

検索対象から除外するフォルダ名を `/` 区切りで入力します。

例:

```text
bin/obj/.git
```

`正規表現` がオンの場合、除外フォルダも正規表現として扱います。

### 除外拡張子

検索対象から除外する拡張子を `/` 区切りで入力します。

例:

```text
exe/dll/pdb
```

先頭の `.` は付けても付けなくてもかまいません。

### 全拡張子を収集

`全拡張子を収集` ボタンを押すと、指定フォルダ配下に存在する拡張子を収集する別画面を開きます。

収集結果は `/` 区切りで表示されるため、除外拡張子の検討に使えます。

### その他のオプション

- `大小区別`: 大文字/小文字を区別して検索します。
- `正規表現`: 検索キーワードを正規表現として扱います。
- `タグジャンプ`: 検索結果をダブルクリックしたとき、サクラエディタで該当行へ移動します。
- `メソッド名を導出`: 検索結果にメソッド名を表示します。現在は Java を主に想定しています。
- `コメント無視`: コメント行を検索対象から除外します。現在は Java を主に想定しています。
- `バイナリファイルを無視`: バイナリと思われるファイルを検索対象から除外します。

## 検索結果一覧

検索結果には以下の列が表示されます。

- `ファイルパス`
- `ファイル名`
- `拡張子`
- `行`
- `Grep結果`
- `メソッド`

`メソッド` 列は `メソッド名を導出` がオンの場合に表示されます。

## 検索結果の絞り込み

検索結果一覧の上にあるフィルタ欄で、表示中の検索結果を絞り込めます。

- ファイルパス
- ファイル名
- 拡張子
- 行
- Grep結果
- メソッド

`クリア` ボタンでフィルタ条件をすべて消去します。

## 検索結果を開く

検索結果の行をダブルクリックすると、該当ファイルを開きます。

`タグジャンプ` がオンの場合:

- サクラエディタが見つかる場合は、該当行へジャンプして開きます。
- サクラエディタが見つからない場合は、通常の関連付けでファイルを開きます。

`タグジャンプ` がオフの場合:

- 該当ファイルのフォルダをエクスプローラーで開きます。

サクラエディタのパスは、`App.config` の `SakuraEditorPath`、標準インストール先、PATH の順に探索します。

## コピー機能

### ファイルコピー

検索結果の行を選択して `ファイルコピー` ボタンを押すと、実在するファイルをクリップボードのファイルドロップ形式でコピーします。

そのままエクスプローラーなどへ貼り付けできます。

### 全てコピー

検索結果一覧を右クリックし、`全てコピー` を選ぶと、表示中の検索結果をタブ区切りテキストとしてクリップボードへコピーします。

## Export sakura

`Export sakura` ボタンを押すと、現在表示中の検索結果を `.grep` ファイルとして出力します。

出力先:

```text
実行ファイルのフォルダ\yyyyMMdd_HHmmssfff.grep
```

出力ファイルの先頭には検索条件が記録されます。記録範囲は以下の行で囲まれます。

```text
# SimpleGrep Search Conditions Begin
# SimpleGrep Search Conditions End
```

この検索条件ブロックは、SimpleGrep に再インポートするときに検索条件の復元へ使われます。検索結果としては扱われません。
検索条件の値は通常そのまま出力されます。改行を含む値など、1 行で安全に書けない値だけ `base64:` 形式で出力されます。

サクラエディタが見つかった場合は、出力後に自動で `.grep` ファイルを開きます。

## Sakura grep ファイルのインポート

`Export sakura` で出力した `.grep` ファイルを検索結果一覧へドラッグ＆ドロップすると、インポート確認ダイアログが表示されます。

- `OK`: 検索結果と検索条件を復元します。
- `キャンセル`: 何もしません。

復元される検索条件:

- 検索フォルダ
- 対象ファイル
- 検索キーワード
- 複数キーワード
- 除外フォルダ
- 除外拡張子
- サブフォルダも対象
- 大小区別
- 正規表現
- メソッド名を導出
- コメント無視
- バイナリファイルを無視

フィルタ条件は export sakura には出力されないため、インポート時にも復元されません。

## 設定のエクスポート

`設定Export` ボタンを押すと、現在の設定を JSON ファイルへ出力します。

出力先:

```text
実行ファイルのフォルダ\SimpleGrep.settings_YYYYMMDD_hhmmss.json
```

通常の自動保存ファイル `SimpleGrep.settings.json` とは別ファイルとして保存されます。

## 履歴

以下の入力欄は履歴を保持します。

- 検索フォルダパス
- 対象ファイル
- 検索キーワード
- 除外フォルダ
- 除外拡張子

履歴は最大10件です。

## 注意事項

- バイナリ判定はファイル先頭付近に NULL バイトがあるかを見て判定します。
- 一部の Java 向け機能は、他言語では期待どおりに動作しない場合があります。
- アクセス権限がないフォルダや読み取りに失敗したファイルは、検索中にスキップまたはエラー表示される場合があります。
- export sakura の検索条件ブロックは SimpleGrep 用のメタ情報です。手編集する場合は、開始行と終了行を削除または変更しないでください。
