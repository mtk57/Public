# SimpleDiff テストデータ

`test/data/src` と `test/data/dst` をアプリの比較元・比較先に指定すると、以下の差分が検出されます。サブフォルダ有効のテストも兼ねています。

| 種別 | パス (相対) | 期待される挙動 |
| --- | --- | --- |
| サイズ差 | `different-size.txt` | ファイルサイズが異なるため差分として表示される |
| 内容差 | `different-content.txt` | サイズは同じだが内容が異なるため差分として表示される |
| 不足 (比較先なし) | `src-only.txt` | 比較先にファイルが無いので差分として表示される |
| 不足 (サブフォルダ) | `nested/src-only-sub.txt` | 比較先にファイルが無いので差分として表示される |
| 内容差 (サブフォルダ) | `nested/different-sub.txt` | サブフォルダ内で内容が異なるため差分として表示される |

下記のファイルは比較結果には現れない（または差分なし）ことを確認します。

- `identical.txt`, `nested/identical-sub.txt`: 内容が完全一致のため差分なし
- `dst-only.txt`: 現仕様では比較元に存在しないファイルは走査対象外のため結果に表示されない

## 使い方
1. SimpleDiff を起動し、`test/data/src` を比較元、`test/data/dst` を比較先に設定します。
2. 「サブフォルダも含める」を ON（既定）で開始します。
3. DataGridView に上記テーブルのファイルが表示され、ダブルクリックで WinMerge から内容差分を確認できます。
